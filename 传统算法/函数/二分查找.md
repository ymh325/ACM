# C++ 中 `lower_bound` 的两种写法

在 C++ 中，`lower_bound` 常见有两种写法：

```cpp
容器.lower_bound(x)
lower_bound(begin, end, x)
```

它们名字很像，但**适用容器、调用方式、效率特点**并不一样。

# 一、先记结论

## 一句话区分

**关联容器用成员函数，顺序容器用算法函数。**

也就是：

1. `set`、`map`、`multiset`、`multimap` 一般写 `s.lower_bound(x)`。
2. 数组、`vector`、`deque` 一般写 `lower_bound(begin, end, x)`。

# 二、对比表

| 特性         | `v.lower_bound(x)`                           | `lower_bound(v.begin(), v.end(), x)`    |
| :--------- | :------------------------------------------- | :-------------------------------------- |
| **适用容器**   | 关联容器（如 `set`, `map`, `multiset`, `multimap`） | 序列容器（如 `vector`, `array`, `deque`）和关联容器 |
| **成员函数  ** | 是容器的成员方法                                     | 是标准库算法函数                                |
| **效率**     | 通常更高效（O(log n)）                              | 对序列容器需要先排序，对关联容器效率较低                    |
| **使用前提**   | 容器本身有序（关联容器自动排序）                             | 范围必须已排序                                 |
| **返回值**    | 迭代器                                          | 迭代器                                     |

# 三、`lower_bound` 的含义

## 基本定义

`lower_bound` 的作用是：

**找到区间中第一个大于等于 `x` 的位置。**

也就是说，若有一个升序序列：

```cpp
1 2 2 4 4 4 7 9
```

那么：

```cpp
lower_bound(..., 4)
```

会返回第一个 `4` 的位置。

```cpp
lower_bound(..., 3)
```

会返回第一个大于等于 `3` 的位置，也就是第一个 `4` 的位置。

如果整个区间都小于 `x`，那么会返回区间末尾，即 `end()`。

## 为什么 `set/map` 更适合用成员函数

对于 `set`、`map` 这一类关联容器：

```cpp
s.lower_bound(x)
```

是容器本身提供的成员函数，它会利用红黑树等底层有序结构直接查找，因此时间复杂度是 $O(log n)$。

而：

```cpp
lower_bound(s.begin(), s.end(), x)
```

虽然语法上也能写，但标准算法版本本质上依赖“迭代器移动”。

对于 `set/map` 的迭代器，它们不是随机访问迭代器，不能像数组那样直接跳到中点，因此实际效率会变差，不适合作为常规写法。

所以结论是：

**关联容器用成员函数。**

**数组、`vector` 这类顺序容器用算法函数。**

# 四、常见容器的写法

### 1、`vector`

```cpp
vector<int> v = {1, 3, 3, 5, 8};
auto it = lower_bound(v.begin(), v.end(), 3);
```

这里 `it` 会指向第一个值为 `3` 的元素。

如果想得到下标：

```cpp
int pos = lower_bound(v.begin(), v.end(), 3) - v.begin();
```

### 2、普通数组

```cpp
int a[] = {1, 3, 3, 5, 8};
int n = 5;
int pos = lower_bound(a, a + n, 3) - a;
```

### 3、`set`

```cpp
set<int> s = {1, 3, 5, 7};
auto it = s.lower_bound(4);
```

这里 `it` 会指向 `5`。

### 4、`map`

```cpp
map<int, string> mp;
mp[2] = "a";
mp[5] = "b";
mp[9] = "c";

auto it = mp.lower_bound(4);
```

这里 `it` 会指向键为 `5` 的键值对。

# 五、常见误区与注意事项

## 常见误区

很多人看到：

```cpp
v.lower_bound(x)
```

会下意识以为 `vector` 也能这样写。

其实不行。

因为 `vector` 没有 `lower_bound` 这个成员函数，必须写成：

```cpp
lower_bound(v.begin(), v.end(), x)
```

也就是说：

```cpp
vector<int> v;
v.lower_bound(3);  // 错误
```

```cpp
set<int> s;
s.lower_bound(3);  // 正确
```

## 使用前提

标准库算法版本：

```cpp
lower_bound(begin, end, x)
```

要求这段区间必须已经按升序排好。

例如：

```cpp
vector<int> v = {5, 1, 3, 2};
auto it = lower_bound(v.begin(), v.end(), 3);
```

这样的结果是**没有意义的**，因为序列无序。

正确写法应先排序：

```cpp
sort(v.begin(), v.end());
auto it = lower_bound(v.begin(), v.end(), 3);
```

而 `set/map` 本身天然有序，所以成员函数不存在这个问题。

## 返回值怎么处理

`lower_bound` 返回的是一个迭代器，因此经常要判断它是否越界。

### 对 `vector` / 数组

```cpp
auto it = lower_bound(v.begin(), v.end(), x);
if(it == v.end()){
	// 没有大于等于 x 的元素
}
else{
	// *it 就是第一个大于等于 x 的值
}
```

### 对 `set`

```cpp
auto it = s.lower_bound(x);
if(it == s.end()){
	// 没找到
}
else{
	// *it 就是答案
}
```

# 六、相关函数对比

## `lower_bound` 和 `upper_bound` 的区别

这两个函数非常容易放在一起考。

`lower_bound(x)`：第一个 **大于等于** `x` 的位置。

`upper_bound(x)`：第一个 **大于** `x` 的位置。

例如序列：

```cpp
1 2 2 2 4 6
```

则：

```cpp
lower_bound(..., 2)  // 指向第一个 2
upper_bound(..., 2)  // 指向 4
```

因此：

```cpp
upper_bound(...) - lower_bound(...)
```

常用于统计某个数出现的次数。

## 典型用法：统计出现次数

对于有序数组或有序 `vector`：

```cpp
int cnt = upper_bound(v.begin(), v.end(), x) - lower_bound(v.begin(), v.end(), x);
```

这表示区间中值为 `x` 的元素个数。

# 七、题目中的常见用途

### 1、查找第一个大于等于 `x` 的位置

```cpp
int pos = lower_bound(a + 1, a + n + 1, x) - a;
```

表示在 `a[1...n]` 中，找到第一个大于等于 `x` 的位置。

### 2、离散化

```cpp
sort(b + 1, b + n + 1);
int m = unique(b + 1, b + n + 1) - (b + 1);
int id = lower_bound(b + 1, b + m + 1, x) - b;
```

用于把原数值映射到离散后的编号。

### 3、最长上升子序列优化

在 LIS 的 $O(nlogn)$ 写法中，也会用到：

```cpp
int pos = lower_bound(b + 1, b + len + 1, x) - b;
```

用 `x` 替换掉第一个大于等于它的位置。

# 八、总结

## 记忆要点

记忆时直接抓住下面几点：

1. `lower_bound` 找的是第一个大于等于 `x` 的位置。
2. `vector`、数组用 `lower_bound(begin, end, x)`。
3. `set`、`map` 用成员函数 `lower_bound(x)`。
4. 标准算法版本要求区间有序。
5. `lower_bound` 和 `upper_bound` 常配合使用，用来求出现次数或边界。
