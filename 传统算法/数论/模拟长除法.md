# 模拟长除法

## 核心思想

**一边用模运算处理"被除数"，一边用长除法逐位还原商**

| 方面 | 传统方法 | 本方法 |
|------|---------|--------|
| 构造 | 先构造完整大数 | 边构造边处理 |
| 存储 | 需要存储第一个N位的 111...1 | 只存余数 |
| 复杂度 | 溢出风险大 | 完全避免 |

---

## 问题定义

你会遇到这样的题目：

> 给定 $x$，找满足以下条件的最小 $n$：
> - $P_n = \underbrace{111...1}_{n个1}$ 
> - $P_n \equiv 0 \pmod{x}$
> - **且需要输出** $P_n / x$ **（即商）**

错误思路：直接算 $P_n \mod x$，然后除
正确思路：在计算 mod 的同时，逐位还原商

---

## 从小学长除法入手

### 例子：$11111 \div 31$

```
     358
   --------
31 | 11111
     93
     ----
      181
      155
      ----
       261
       248
       ----
        13
```

**每一步的三个操作：**
1. `当前数 = 上一步余数 * 10 + 新一位`
2. `商的一位 = 当前数 / x`
3. `新余数 = 当前数 % x`

---

## 核心代码转移

### 关键发现

如果你会：
```cpp
cur = (cur * 10 + 1) % x;  // 只记录余数
```

那么长除法只是把它**拆开**就行：

```cpp
cur = cur * 10 + 1;        // 当前被除数
int digit = cur / x;       // 商的这一位
cur %= x;                  // 更新余数
```

**本质一样，只是多拿了商的每一位！**

---

## 完整步骤详解（以 $x=31$ 为例）

### 初始状态
```
cur = 0
s = ""
```

### 第 1 位
```
cur = 0 × 10 + 1 = 1
digit = 1 ÷ 31 = 0，余 1
s = "0"
cur = 1
```

### 第 2 位
```
cur = 1 × 10 + 1 = 11
digit = 11 ÷ 31 = 0，余 11
s = "00"
cur = 11
```

### 第 3 位
```
cur = 11 × 10 + 1 = 111
digit = 111 ÷ 31 = 3，余 18
s = "003"
cur = 18
```

### 第 4 位
```
cur = 18 × 10 + 1 = 181
digit = 181 ÷ 31 = 5，余 26
s = "0035"
cur = 26
```

### 第 5 位
```
cur = 26 × 10 + 1 = 261
digit = 261 ÷ 31 = 8，余 13
s = "00358"
cur = 13
```

**继续这个过程直到余数重复（周期出现）**

最终得到：
```
s = "003584229390681"
去掉前导0：s = "3584229381"
```

## 完整代码框架

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int x;
    int n;
    cin >> x >> n;
    
    int cur = 0;
    string s = "";
    
    // 生成 n 位的 111...1 并做长除法
    for (int i = 1; i <= n; i++) {
        cur = cur * 10 + 1;      // 加一位 1
        int digit = cur / x;     // 商的一位
        cur %= x;                // 余数
        s += char('0' + digit);
    }
    
    // 去掉前导零
    int start = 0;
    while (start < s.length() && s[start] == '0') {
        start++;
    }
    
    cout << s.substr(start) << endl;
    
    return 0;
}
```

## 应用场景识别

你需要用这个方法的**特征信号**：

| 特征 | 说明 |
|------|------|
| 构造超大整数 | 位数达到 $10^3 \sim 10^6$ 级别 |
| 只有小的除数 | $x$ 通常 $< 10^6$ |
| 要求整除 | $P_n \equiv 0 \pmod{x}$ |
| 需要输出商 | 不仅要找 $n$，还要输出 $P_n / x$ |

**识别规则：** 出现以上 2+ 个特征 → 99% 是这类题

### 更复杂的变体

> 找**最小的 01 串**被 $x$ 整除

这需要用 **BFS + 模运算状态**：

```cpp
// 状态 = 当前余数
// 边 = 在该余数状态下，往后面加 0 或 1
queue<int> q;
q.push(0);
// ... BFS构造最短路径
```

这是 ICPC 常客问题，本质是在"余数图"上找最短路径。

## 常见错误速避

| 错误 | 错误做法 | 正确做法 |
|------|---------|----------|
| 数字溢出 | `cur` 用 int64 | 用 `% x` 控制范围 |
| 前导零 | 直接输出 | substr 去掉前导 0 |
| 逻辑混淆 | `digit = cur % x` | `digit = cur / x` |
| 余数更新 | 忘记 `cur %= x` | 每次必须更新 |

---

## 验证方法

对构造出的商进行**反向验证**：

```cpp
// 验证: quotient * x == 111...1
bool verify(string quotient, int x, int n) {
    // 反向乘法：逐位计算 quotient * x
    int carry = 0;
    for (int i = n - 1; i >= 0; i--) {
        int prod = (quotient[i] - '0') * x + carry;
        if (prod % 10 != 1) return false;  // 应该都是 1
        carry = prod / 10;
    }
    return carry == 0;
}
```
