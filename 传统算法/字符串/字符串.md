# 学习字符串的方法论

- 先判断匹配类型
	- 单个模式串匹配
	- 多个模式串匹配
	- 子串比较
	- 前缀查询
	- 字典序问题

- 再看是否需要预处理
	- next / pi 数组
	- Trie 节点
	- fail 指针
	- 前缀哈希

- 再看复杂度要求
	- 暴力枚举是否会超时
	- 字符集大小是多少
	- 字符串总长度是多少
	- 查询次数是多少

- 再看冲突和边界
	- 哈希冲突
	- 空串
	- 重复模式串
	- 下标从 0 还是 1 开始

- 最后看输出目标
	- 是否只判断存在
	- 是否统计次数
	- 是否输出位置
	- 是否需要最长 / 最短匹配


# 刷题时需要想什么

- 是匹配、比较、统计，还是构造字符串？
- 模式串是一个还是多个？
- 是否需要用前缀函数维护失配后的跳转？
- 是否需要 Trie 维护公共前缀？
- 多模式匹配是否需要 AC 自动机？
- 子串相等能否用哈希快速判断？
- 字符集是小写字母、ASCII，还是更大范围？
- 下标和长度的转换是否统一？
