# 输入输出

在算法比赛以及平时做题的过程中，我们会有输入输出，并且有**两种方式可以选择**；
一种是C++的，一种是C语言的。

- C++的标准输入输出流是 **`cin/cout`**；
	
- C语言的是 **输入** `scanf()` / **输出** `printf()`；

> [!important] 
> 重点要说的是 在一般情况下 **C语言的输入输出是比 C++快的**，以下原因了解一下即可
> 1. **默认同步**：C++为了与C的`stdio`兼容，默认同步了`iostream`和`stdio`，带来额外开销。
> 2. **格式化方式**：C的`printf`/`scanf`使用编译时解析的格式字符串，效率更高；而`cin`/`cout`依赖运行时类型推断和操作符重载，开销较大。
> 3. **绑定刷新**：`cout`默认与`cin`绑定，每次输入可能刷新输出缓冲区，影响性能

然后，我们要知道的是，**如何让C++的输入输出速度接近C语言，** 因为 `cin/cout` 应该是比 `printf`/`scanf` 更顺手一点，如果各位同学觉得两种写起来都差不多那直接用C语言的写法也是可以的。

需要添加如下代码：
1. **关闭同步**：在main函数开头添加 `ios::sync_with_stdio(false);` 
    这会断开C++标准流与C标准流的同步，减少开销。

2. **解除绑定**：`cin.tie(nullptr);` / `cin.tie(0);`
    默认情况下，`cin` 和 `cout` 绑定，每次 `cin` 操作前会刷新 `cout` 缓冲区。解除绑定可避免这种刷新。
3. **避免使用 `endl`** ：可以用 `'\n'` 代替 `endl`，因为 `endl` 会额外刷新缓冲区。

> [!warning] 
> 在没有关闭同步时，C++和C的输入输出是可以混用的。
> 关闭了过后，就不能混用，只能用其中一种！！！


## 具体代码写法如下：

```cpp
int main(){
	ios::sync_with_stdio(false);cin.tie(0);
	
	cout<<1<<'\n'; // 这里换行直接写'\n'即可
	
	return 0;
}
```

**或者**

可以在 `main` 外面写一个宏定义，可能会美观一点
```cpp

#define OFF ios::sync_with_stdio(false);cin,tie(0);
#define endl '\n'

int main(){
	OFF // 直接引用宏定义关闭同步
	
	cout<<1<<endl;
	return 0;
}
```



# 变量的定义

这个也是有门道的。

## 普通变量的定义

把一个基础变量定义在 `main` 里面和 `main`  外是有区别的。

> [!NOTE] 官方的表述
> 在 C++ 中，将变量定义为**全局变量**（在函数外部定义）或在**命名空间作用域**中定义时，这些变量具有**静态存储期**（static storage duration）。

- 如果变量具有**基本类型**（如 `int`、`float`、`double`、`char` 等），并且没有显式初始化
  
- 那么它们会执行**零初始化**（zero-initialization）
  
- 即被初始化为对应类型的零值：
  
    - `int` → `0`
      
    - `float`/`double` → `0.0`
      
    - `bool` → `false`
      
    - 指针 → `nullptr`
      
    - `char` → `'\0'`

```cpp
#include <iostream>
using namespace std;

// 全局变量 - 具有静态存储期
int global_int;      // 零初始化为 0
double global_double; // 零初始化为 0.0
bool global_bool;    // 零初始化为 false
int* global_ptr;     // 零初始化为 nullptr

int main() {
    cout << "global_int: " << global_int << endl;       // 输出 0
    cout << "global_double: " << global_double << endl; // 输出 0
    cout << "global_bool: " << global_bool << endl;     // 输出 0 (false)
    cout << "global_ptr: " << global_ptr << endl;       // 输出 0 (nullptr)
    
    // 局部变量 - 具有自动存储期，不会自动初始化
    int local_int;
    cout << "local_int: " << local_int << endl;  // 未定义行为，可能是任意值
    
    return 0;
}
```


## 数组的定义

在**全局定义数组**和**局部定义**也有一些差别。

<font color="#f79646">算法竞赛</font>中主要考虑如下：

1. **大小限制**：
	- **全局数组**：可以定义**非常大**的数组（受限于可用内存）
		
	- **局部数组**：大小**受限**（栈空间有限，通常约1-8MB，容易栈溢出）
2. **初始化**：
	- **全局数组**：**自动初始化**为零值（零初始化）
	  
	- **局部数组**：**不会自动**初始化（值是未定义的随机值）


> [!NOTE]
> 所以，如果使用数组的话，可以把数组定义在全局，也就是 `main` 外面，并且定义一个常量作为这个全局数组的大小
>

**示例代码**：
```cpp
#include <iostream>
using namespace std;
const int N = 1e5+5; // 1e5的意思是10^5
int a[N];

int main(){
	...
}
```




# 结构体

有时候，我们可能会有将多个变量绑定在一起的要求，这个时候可以使用结构体，应该是最简便的。
关于结构体的基础使用，请各位同学自行在B站进行学习。


此处仅讲解如何用 `sort` 对结构体变量进行排序；

有很多方式，这里我<font color="#f79646">最推荐两种</font>：

- **使用自定义比较函数**
- **重载结构体的比较运算符**



## 自定义比较函数

- **单变量比较**，就是在比较的时候只考虑结构体的其中一个变量
```cpp
#include <algorithm>
#include <iostream>
using namespace std;

struct Student {
    string name;
    int score;
    int age;

};

// 比较函数：按分数降序
bool ByScore(const Student &a, const Student &b) {
    return a.score > b.score;  // 降序
}

// 比较函数：按年龄升序
bool ByAge(const Student &a, const Student &b) {
    return a.age < b.age;  // 升序
}

int main() {

    Student stu[2] = {{"Alice", 85, 20}, {"Bob", 92, 19}};
    
    sort(stu, stu + 2, ByScore);

    // 或

    sort(stu, stu + 2, ByAge);

    return 0;

}
```

- **多变量比较**，就是考虑结构体中的多个变量
```cpp
// 先按分数降序，分数相同按年龄升序
bool Multi(const Student &a, const Student &b) {
    if (a.score != b.score)
        return a.score > b.score;  // 分数高的在前
    return a.age < b.age;  // 分数相同则年龄小的在前
}

// 使用
sort(stu, stu+2, Multi);
```



## 重载结构体的比较运算符

就是说，我们会在结构体的**内部**定义好**结构体的 `<` 和 `>` 等比较运算符的规则**。

例如：
```cpp
struct Student {
    string name;
    int score;
    
    // 重载 < 运算符（用于sort的默认比较）
    bool operator<(const Student &p) const {
        return score < p.score;  
    }
    // 重载 > 运算符（用于sort的升序比较)
	bool operator>(const Student &p) const {
        return score > p.score;  
    }
};

// 使用默认sort，依靠 < 规则排序
sort(stu.begin(), stu.end()); // 注意这里的stu是用vector存储的，所以有.begin()和.end()的用法
// 依靠 > 排序
sort(stu.begin(), stu.end(), greater<Student>()); 
```



# while迭代

对于一个变量 `t` ，如果我们要循环 `t` 次，并且没有其他作用了，那么可以使用如下的写法

```cpp
while(t--){
	...
	...
}
```

这也是迭代 `t` 次，结束后 `t` 会变成 -1；





# 不考虑类与对象

在算法比赛中，基本不会用到类，一般都用结构体，所以准备算法比赛不用学类，只需掌握结构体的基本使用与排序即可


 **核心区别**

| 特性         | **struct（结构体）** | **class（类）** |
| ---------- | --------------- | ------------ |
| **默认访问权限** | `public`        | `private`    |
| **默认继承方式** | `public`        | `private`    |
| **使用习惯**   | 简单数据聚合          | 复杂对象封装       |
| **C兼容性**   | 支持（有限）          | 不支持          |
