# 学习图论的方法论

- 先判断图的性质
	- 有向图还是无向图
	- 有权图还是无权图
	- 是否连通
	- 是否有环
	- 是否是树 / DAG / 基环树

- 再看题目目标
	- 可达性
	- 最短路
	- 连通性
	- 最小生成结构
	- 路径查询
	- 依赖顺序

- 再看边权条件
	- 无权边用 BFS
	- 非负边权考虑 Dijkstra
	- 有负边考虑 Bellman-Ford / SPFA
	- 多点互相距离考虑 Floyd 或多源最短路

- 再看复杂度
	- n 和 m 的规模
	- 稠密图还是稀疏图
	- 单次询问还是多次询问
	- 是否能预处理

- 最后看建图
	- 点表示什么状态
	- 边表示什么转移
	- 边权表示什么代价
	- 是否需要反向边、虚拟源点、分层图


# 刷题时需要想什么

- 题目是否本质上是建图问题？
- 点、边、边权分别代表什么？
- 图是否有特殊性质：树、DAG、基环树、二分图？
- 最短路是否有负边，是否可能出现负环？
- 多次查询是否需要 LCA、树剖或预处理距离？
- 拓扑顺序是否能把问题转成 DP？
- 无向图中的边是否需要开两条？
- 是否需要处理不连通图？
