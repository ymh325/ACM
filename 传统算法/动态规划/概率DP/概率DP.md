# 概率DP

> [!quote]
> 利用动态规划的思想去解决概率、期望等题目


**类型分两类**：
- 求概率，常用 **正推**，即由起始状态到终止状态枚举
	
- 求期望，常用 **倒推**，即由终止状态到起始状态枚举 

---

## 概率

**概率**：随机事件发生的可能性，是一个0到1之间的实数。

### 古典模型的特点

1. **有限性**：所有可能出现的基本事件只有有限个
    
2. **等可能性**：每个基本事件出现的可能性相等
    

例如，掷骰子，六个面，每个面朝上的概率为 $\frac{1}{6}$

### 计算概率

- **分类**用加法原理
    
- **分步**用乘法原理
    

### 概率公式

| 事件   | 概率公式                                      | 条件/说明       |
| ---- | ----------------------------------------- | ----------- |
| A    | $P(A) \in [0,1]$                          | 基本概率        |
| 非A   | $P(\overline{A}) = 1 - P(A)$              | 对立事件        |
| A或B  | $P(A \cup B) = P(A) + P(B) - P(A \cap B)$ | 一般情况        |
| A或B  | $P(A \cup B) = P(A) + P(B)$               | $A, B$ 互斥事件 |
| A和B  | $P(A \cap B) = P(A)P(B\|A)=P(B)P(A\|B)$   | 一般情况        |
| A和B  | $P(A \cap B) = P(A)P(B)$                  | $A, B$ 独立事件 |
| 条件概率 | $P(A\|B) = \frac{P(A \cap B)}{P(B)}$      | $P(B) > 0$  |
|      |                                           |             |

## 期望


### 期望的定义

设离散型随机变量 $X$ 有 $n$ 种取值，第 $i$ 种取值为 $Xi$​，出现的概率是 $pi$​。
则 $X$ 的数学期望 $E(X)=\sum_{i=1}^n​Xi​∗pi$​，即 $X$ 的加权平均值。

**示例**：
$X={2,3,5}，p={21​,31​,61​}$，则 $E(X)=2∗21​+3∗31​+5∗61$​。

### 期望的性质

1. $E(aX+b)=aE(X)+b$，$a$，$b$ 是常数
2. $E(X+Y)=E(X)+E(Y)$
3. $E(XY)=E(X)E(Y)$，$X$，$Y$ 相互独立

### 全期望公式

当 $Y$ 是**离散随机变量**，取值于可数集 $\{y_1,y_2,… \}$ 时，条件期望 $E[X∣Y]$ 在 $\{ Y=y_i\}$ 上取常数值  $E[X∣Y=yi]$ 。于是全期望公式化为：

$$
E[X]=\sum_{i}^{n}E[X∣Y=y_i]⋅P(Y=y_i)
$$

# 正推与逆推

> [!question]
> 现在，讲一讲为什么 **概率是正推，期望是逆推**

## 正推

先说概率的正推吧，这个好理解一些；


> [!abstract] 
>
**概率的正推 =**
>> 已知 ” 现在发生了什么”，  
>> 问 “ 下一步会发生什么，以及概率是多少 ”

也就是：
$$
P(下一步是A|现在是B)
$$
#### 举例：抛硬币
你现在要抛一枚**均匀硬币**。
我们能问的问题只有一种是**天然的**：

- 下一次是正面还是反面？
- 概率各是多少？
$$\Pr(\text{正面})=\frac12,\quad \Pr(\text{反面})=\frac12​$$
这就是**正推**。

---

> [!error] 
> 我们不能自然的问
> >“为了最终抛出正面，这一次应该以多大概率是反面？”
> 这个问题本身是**没有时间顺序意义的**。


概率不是结果的属性，而是：

> **在事件发生之前，对不确定性的度量**

所以概率永远是：

- **现在 → 未来**
    
- **已知 → 未知**

这在逻辑上是单向的。

---
## 逆推


> [!abstract]
**期望的逆推 =**
> > 已知 **“终点的代价是多少”**，  
> > 问 **“从现在出发，平均还要付出多少总代价”**

也就是：  
$$  
E(\text{总代价}\mid \text{现在是 }B)  
$$

---


## 期望为什么必须逆推？

因为：

> **期望描述的是“未来所有步骤的总和”**

而“总和”这个概念，**必须先知道什么时候结束**，才能定义。

所以期望永远是：

- **未来 → 现在**
    
- **终点 → 起点**
    

---

### 再换一种说法

- **概率**：  
    描述“世界如何一步一步运行”  
    👉 生成规则 → 正推
    
- **期望**：  
    描述“走完整个过程要花多少代价”  
    👉 结果函数 → 逆推
    

---

## 对照表

|对象|概率|期望|
|---|---|---|
|描述内容|下一步发生什么|到终点还要多少|
|是否依赖终点|否|是|
|是否累计|否|是|
|推导方向|正推|逆推|
|本质|生成规则|解方程|

---

> [!summary]  
> **概率是“怎么走”的规则，只能向前；  
> 期望是“走完要多久”的结果，只能从终点往回算。**

## 求期望

### Te1 [POJ2096 Collecting Bugs](http://poj.org/problem?id=2096)

#### 问题描述

一个软件有 `s` 个子系统，会产生 `n` 种 bug。

某人一天发现一个 bug，这个 bug 属于一个子系统，属于一个分类。

每个 bug 属于某个子系统的概率是 `1/s`，属于某种分类的概率是 `1/n`。

问：发现 `n` 种 bug，且 `s` 个子系统都发现 bug 的期望天数（`0 < n, s <= 1000`）。

---

#### 状态定义

- 状态：`f[i][j]` 表示已经找到 `i` 种 bug，`j` 个子系统的 bug，达到目标状态的**期望天数**。
- 终点：`f[n][s] = 0`
- 答案：`f[0][0]`

---

#### 状态转移 
考虑 `f[i][j]` 的状态转移：

 1. $f[i][j]$：发现一个 bug 属于已有的 `i` 个分类、`j` 个系统，
	概率为 $p_1​=\frac in⋅ \frac js$​
	 
 2. $f[i][j+1]$：发现一个 bug 属于已有的分类，不属于已有的系统，
	概率为 $p_2​=\frac in​⋅(1−\frac js​)$
	
 3. $f[i+1][j]$：发现一个 bug 不属于已有的分类，属于已有的系统，
	概率为 $p_3​=(1−\frac in​)⋅ \frac js​$
	
 4. $f[i+1][j+1]$：发现一个 bug 不属于已有的分类，不属于已有的系统，
	概率为 $p_4​=(1−\frac in​)⋅(1− \frac js​)$ 

---
#### 期望方程


> [!todo] 
> 接下来就是，写出期望方程，也就是递推公式

**首先**，我们要知道 $f[i][j]$ 所代表的就是一个期望值，所以计算 $f[i][j]$ 其实就是在计算期望；

**但是要注意**，我们在进行DP递推的时候，$f[i][j]$ 最好是取决于其他的 $f[x][y]$ 值，因为我们只有 `f[][]`  的值是可以确定的，又因为`f[][]` 本身是一个期望值，所以我们需要一个**期望与期望想关联的公式**；

**那么**，我们就把全期望公式抬出来：
$$
\begin{align*}
E[X] &=\sum_{i}^{n}E[X∣Y=y_i]⋅P(Y=y_i) \\
\because f[i][j] &= E[X]\\
\therefore \ f[i][j] &= \sum_{i}^{n}E[X∣Y=y_i]⋅P(Y=y_i)
\end{align*}
$$
根据上述的状态转移方程，每到新的一天时，会消耗 1 代价
- 当 $Y=y_1$时 ，$f[i][j]=f[i][j]+1$ ，$p_1​=\frac in⋅ \frac js$
	
- 当 $Y=y_2$时 ，$f[i][j]=f[i+1][j]+1$ ， $p_2​=\frac in​⋅(1−\frac js​)$
	
- 当 $Y=y_3$时 ，$f[i][j]=f[i][j+1]+1$ ，$p_3​=(1−\frac in​)⋅ \frac js​$
	
- 当 $Y=y_4$时 ，$f[i][j]=f[i+1][j+1]+1$ ，$p_4​=(1−\frac in​)⋅(1− \frac js​)$ 

所以，根据 **全期望公式** ，得如下：

$$
\begin{align*}
   f[i][j]
		&=(f[i][j]+1)\cdot p_1 +(f[i][j+1]+1) \cdot p_2 +(f[i+1][j]+1)\cdot p_3 + (f[i+1][j+1]+1)\cdot p_4 \\
		&= f[i][j]⋅p_1​+f[i][j+1]⋅p_2​+f[i+1][j]⋅p_3​+f[i+1][j+1]⋅p_4 + p_1 +p_2+p_3+p_4\\
		&= f[i][j]⋅p_1​+f[i][j+1]⋅p_2​+f[i+1][j]⋅p_3​+f[i+1][j+1]⋅p_4​+1 \\
        &= \frac{(f[i][j+1]⋅p_2​+f[i+1][j]⋅p_3​+f[i+1][j+1]⋅p_4​+1)}{1-p_1}  \\
        &= \frac{f[i][j+1] \cdot i \cdot (s-j)
                    +f[i+1][j] \cdot (n-i) \cdot j
                    +f[i+1][j+1]\cdot (n-i)\cdot (s-j)+n \cdot s} 
                     {n \cdot s-i \cdot j}
\end{align*}

$$

#### 代码

```cpp
#include <iostream>
using namespace std;
#define OFF ios::sync_with_stdio(0),cin.tie(0);
// #define int long long
const int N = 1e3 + 5;
int n,s;
double f[N][N];
int main(){
    cin>>n>>s;
    f[n][s] = 0;
    for(int i=n;i>=0;i--){
        for(int j=s;j>=0;j--){
            if(i==n&&j==s)continue;
            f[i][j]=(f[i][j+1]*i*(s-j)
                    +f[i+1][j]*(n-i)*j
                    +f[i+1][j+1]*(n-i)*(s-j)+n*s) / (n*s-i*j);
        }
    }
    printf("%.4f",f[0][0]);
    return 0;
}
```
