以下是 C++ `std::vector` 的常用操作及其时间复杂度总结。`vector` 是基于动态数组实现的顺序容器，支持随机访问，在尾部增删效率高，但在中间或头部增删需要移动元素。

---

### 1. 构造与赋值

| 操作 | 说明 | 时间复杂度 |
|------|------|-------------|
| `vector<T> v` | 默认构造空容器 | O(1) |
| `vector<T> v(n)` | 构造包含 n 个值初始化元素的容器 | O(n) |
| `vector<T> v(n, val)` | 构造包含 n 个值为 val 的元素的容器 | O(n) |
| `vector<T> v(beg, end)` | 迭代器范围构造 | O(n) (n = 距离) |
| `vector<T> v(initializer_list)` | 列表初始化 | O(n) |
| `v.assign(n, val)` | 赋值为 n 个 val | O(n) |
| `v.assign(beg, end)` | 赋值为迭代器范围内容 | O(n) |
| `v = other` / `v = { ... }` | 拷贝/移动/列表赋值 | O(n) / O(1) (移动) |

---

### 2. 元素访问（边界未检查）

| 操作          | 说明          | 时间复杂度 |
| ----------- | ----------- | ----- |
| `v[i]`      | 下标运算符，无边界检查 | O(1)  |
| `v.at(i)`   | 带边界检查（抛异常）  | O(1)  |
| `v.front()` | 第一个元素引用     | O(1)  |
| `v.back()`  | 最后一个元素引用    | O(1)  |
| `v.data()`  | 返回底层数组指针    | O(1)  |

---

### 3. 迭代器

| 操作 | 说明 | 时间复杂度 |
|------|------|-------------|
| `v.begin() / end()` | 获取首/尾迭代器 | O(1) |
| `v.cbegin() / cend()` | 常迭代器 | O(1) |
| `v.rbegin() / rend()` | 反向迭代器 | O(1) |

---

### 4. 容量与大小

| 操作 | 说明 | 时间复杂度 |
|------|------|-------------|
| `v.empty()` | 是否为空 | O(1) |
| `v.size()` | 元素个数 | O(1) |
| `v.capacity()` | 当前已分配空间大小 | O(1) |
| `v.max_size()` | 理论上能容纳的最大元素数 | O(1) |
| `v.reserve(n)` | 预留至少 n 个元素的存储空间 | O(n) (若需重新分配，拷贝/移动现有元素) |
| `v.shrink_to_fit()` | 释放多余容量 | O(n) (可能重新分配) |
| `v.resize(n)` / `v.resize(n, val)` | 改变元素数量，新增元素值初始化或复制 val | O(n) |

---

### 5. 修改操作（增删改）

| 操作                             | 说明               | 时间复杂度                |
| ------------------------------ | ---------------- | -------------------- |
| `v.push_back(val)`             | 尾部插入元素           | 摊销 O(1)              |
| `v.pop_back()`                 | 删除尾部元素           | O(1)                 |
| `v.insert(pos, val)`           | 在 pos 前插入一个元素    | O(n) (移动 pos 及之后元素)  |
| `v.insert(pos, n, val)`        | 插入 n 个 val       | O(n + k) (k = 插入数量)  |
| `v.insert(pos, beg, end)`      | 插入迭代器范围          | O(n + m)             |
| `v.erase(pos)`                 | 删除 pos 处的元素      | O(n)                 |
| `v.erase(beg, end)`            | 删除 [beg, end) 范围 | O(n)                 |
| `v.clear()`                    | 清空所有元素           | O(n) (元素析构)          |
| `v.emplace(pos, args...)`      | 在 pos 前原位构造元素    | O(n)                 |
| `v.emplace_back(args...)`      | 尾部原位构造           | 摊销 O(1)              |
| `swap(v1, v2)` / `v1.swap(v2)` | 交换内容             | O(1) (仅交换指针，C++11 后) |

---

### 6. 其它常用操作

| 操作 | 说明 | 时间复杂度 |
|------|------|-------------|
| `v == v2` 等关系运算符 | 按字典序比较 | O(n) |
| `std::sort(v.begin(), v.end())` | 排序 | O(n log n) |
| `std::find(v.begin(), v.end(), val)` | 线性查找 | O(n) |
| `std::reverse(v.begin(), v.end())` | 反转 | O(n) |

---

### 复杂度总结要点

- **随机访问**：O(1) — 核心优势  
- **尾部增删**：摊销 O(1)（仅在容量不足时触发扩容，拷贝/移动旧元素 O(n)）  
- **中间或头部增删**：O(n) — 需要移动后续所有元素  
- **扩容机制**：通常按 2 倍或 1.5 倍增长，重新分配时所有元素被拷贝/移动（C++11 后优先移动）
