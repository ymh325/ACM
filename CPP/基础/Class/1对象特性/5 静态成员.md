### 静态成员

静态成员就是在成员变量和成员函数前加上关键字static，称为静态成员

> 注意：静态成员有访问权限

静态成员分为：

- 静态成员变量

  - 所有对象共享同一份数据

  - 在编译阶段分配内存

  - **类内声明，类外初始化**（必须在类外初始化！！！）

- 静态成员函数
  - 所有对象共享同一个函数
  - **静态成员函数只能访问静态成员变量**



静态成员和静态函数都有两种访问方式

```c++
class per(){
public:
    int b;
	static int a;
    static void func(){
        a = 10;
       //b = 10;
       //会报错，静态成员函数 无法直接访问 非静态成员变量
       //需要传入一个具体的类成员
    }
    //例如:
    static void func(per &p){
       p.b = 10;
       //此时不会报错
    }
    
};
//类外部初始化a
int per::a = 1;

int main(){
    
   //1.通过对象访问
    per p;
    cout<<p.a<<endl;
    p.func();
    
   //2.通过类名访问
   	cout<<per::a<<endl;
    per::func();
    
    return 0;
}

```

> 静态成员函数不能访问非静态成员变量的原因
>
> 1. 静态成员函数的生命周期和存储位置
>    - 静态成员函数属于类本身，而不是某个具体的对象实例。它在程序启动时就被加载到内存中，并且在整个程序运行期间都存在。
>    - 静态成员函数的地址是固定的，不依赖于任何对象实例。
> 2. 非静态成员变量的生命周期和存储位置
>    - 非静态成员变量属于类的每个具体对象实例，只有当对象被创建时，这些变量才会被分配内存。
>    - 每个对象实例的非静态成员变量存储在各自的堆栈或堆内存中，彼此独立。
> 3. 调用静态成员函数时缺乏对象上下文
>    - 静态成员函数可以通过类名直接调用，而无需创建对象实例。例如：ClassName::StaticFunction()。
>    - 由于没有具体的对象实例参与调用，静态成员函数无法确定应该访问哪个对象的非静态成员变量。
> 4. 内存模型冲突
>    - 静态成员函数在内存中是全局共享的，而非静态成员变量是对象私有的。
>    - 如果允许静态成员函数访问非静态成员变量，则需要明确指定一个对象实例作为访问目标，但这与静态成员函数的设计初衷相违背（即不依赖于具体对象）。
> 5. 解决方法
>    - 如果需要访问非静态成员变量，可以通过将对象实例作为参数传递给静态成员函数来实现。
