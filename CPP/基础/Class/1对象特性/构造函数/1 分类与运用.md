### 1.1 构造函数的分类与调用

两种分类方式:

​	按参数分类：无参构造和有参构造

​	按类型分类：普通构造和拷贝构造

三种调用方式：

- 括号法：

```c++
per a(10);//per a(1,2,3,4,...)多参数传递
per a(b);
```

- 显示法：

```c++
per a = per(10);//per a = per(1,2,3,4,...)多参数传递
per a = per(b);
```

- 隐式转换法：

```c++
per a = 10;//per a = {1,2,3,4,...}多参数传递
per a = b;
```



### 1.2 拷贝构造函数的调用时机

三种情况：

- 使用一个已经创建完毕的对象来初始化一个新对象

```c++
per a(b)
```

- 值传递的方式给函数参数传值（）

```c++
void test(per a)
```

- 以值方式返回局部对象

```c++
per test(){
	per p1;
	return p1;
}
per p2 = test();
```

> **注意**
>
> 在test()函数中，**p1是一个局部对象。**
> **理论上来说**，当p1被返回时，编译器会创建一个临时对象，并调用拷贝构造函数将p1的内容复制到这个临时对象中，然后再将这个临时对象赋值给p2。
> **然而**，为了优化性能，**编译器可能会直接将p2的内存空间分配给p1**，这样就避免了额外的拷贝操作。因此，p1和p2实际上指向的是同一块内存区域，所以它们的地址相同。
> 标准行为：根据C++标准，编译器被允许进行这种优化，并且不会改变程序的行为。也就是说，即使进行了RVO，程序的输出结果仍然是正确的。
>
> 所以，**在开启编译器优化的情况下，return p1确实没有调用拷贝构造函数**，**而是直接将内存空间分配给了p2**。这种优化技术称为返回值优化（Return Value Optimization, RVO）。



**手动定义拷贝构造函数，才会触发RVO机制**



### 1.3 构造函数调用规则

默认情况下C++编译器至少给一个类添加3个函数

1. 默认构造函数（无参数，函数体为空）
2. 默认析构函数（无参，函数体为空）
3. 默认拷贝函数，对属性进行值拷贝



构造函数调用规则如下：

- 如果用户定义有参构造函数，C++不在提供无参构造，但会提供有参构造
- 如果用户定义拷贝构造函数，C++不在提供其他构造函数
