### 赋值运算符

> 必须声明为成员函数

C++编泽器至少给一个类添加4个函数

1.默认构造函数（无参，函数体为空）

2.默认析构函数（无参，函数体为空)

3.默认拷贝构造函数，对属性进行值拷贝

4.赋值运算符operator:=,对属性进行值拷贝

如果类中有属性指向堆区，做赋值操作时也会出现深浅拷贝问题



```cpp
#include "iostream"
using namespace std;
class per{
public:
    int *ma;
    per (){ ma= new int (0);}
    explicit per(int a){ma=new int(a);}
    per(const per &p){
        ma=p.ma;
        cout<<"copy"<<endl;
    }
    
    //这里的返回值应是引用,才能p1=p2=p3,
    per& operator=(const per &p){
        if (this==&p)return *this;
        
        //先释放掉 ma 原来的内存
        if (ma!= nullptr){
            delete ma;
            ma= nullptr;
        }
    	//这里创建一个值为 *p.ma 的新的内存，
        if (p.ma!= nullptr)ma=new int (*p.ma);
        return *this;
    }
    ~per(){
        delete ma;
    }
};
ostream& operator<<(ostream &out,const per &p){
    out<<p.ma;
    return out;
}
int main() {
    per p(1);
    per p1;
    cout<<p.ma<<endl;
    cout<<*p.ma<<endl;
    p1=p;
    cout<<p1.ma<<endl;
    cout<<*p1.ma<<endl;
    return 0;
}
```



在 C++ 中，重载赋值 `=` 操作符时应声明为成员函数，主要有以下原因：

- **避免二义性**：如果程序员没有为类定义赋值操作符重载函数，编译器会隐式定义一个。此时若在类外再将赋值运算符重载为全局函数，就会和编译器生成的默认赋值运算符冲突，产生二义性 。
- **访问类成员**：赋值操作通常需要访问类的私有和受保护成员，以便能够正确地执行赋值操作，只有成员函数和友元函数可以访问类的私有成员，但赋值操作符重载不适宜用友元函数。
- **语法规则和类型安全**：成员运算符函数的一个特点是类的 `this` 指针会被绑定到运算符的左侧运算对象，使得左侧操作数必须是该类类型的对象。若赋值操作符重载为全局函数（友元函数），就没有了左操作数是 `this` 指针的限制，可能会出现如 `6 = c` （`6` 为常量，`c` 为类对象）这样不符合语法规则的代码。而且，当类存在转换构造函数时，还可能因隐式类型转换导致调用错误 。
- **符合约定和最佳实践** ：C++ 规定赋值（`=`）、下标（`[]`）、调用（`()`）和成员访问箭头（`->`）等运算符必须被定义为类成员操作。将赋值操作符重载为成员函数，是 C++ 编程中的约定和最佳实践，有助于增强代码的可读性和可维护性，也让其他开发者更容易理解代码逻辑 。
