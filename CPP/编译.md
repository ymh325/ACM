# 基础知识

在 C++ 中，从代码到程序要经历四个阶段：

1. **预处理**：处理宏定义和 `#include`。
    
2. **编译**：将源文件翻译成汇编代码。
    
3. **汇编**：将汇编代码转换成机器码，生成 **.o 文件**。
    
4. **链接**：将多个 **.o 文件** 和库文件打包在一起，生成**最终的可执行文件**。



> [!info] 
> 然后，我们要知道的是，头文件是**不直接**参与编译的，在预编译的时候，头文件的所有内容会被复制在被导入的地方


# 命令的运行逻辑



我们来看一个简单的命令
```bash
gcc main.c -o main
```
这个命令会生成 `main.exe` 可执行程序，但是好像并看到什么 `.o` 文件；
实际上，当你运行 `gcc main.c -o main` 时，`gcc` 把刚才提到的预处理、编译、汇编和链接这四个阶段**一气呵成**地执行完了。

如果你的文件夹结构如下：
```text
|--header.h
|--header.cpp
|--main.cpp
```

那么生成可执行文件 `main` 的命令如下：
```bash
g++ main.cpp header.cpp -o main
```

同样的，不会有 `main.o` 、`header.o` 产生。

---

**那如果需要`main.o` 和 `header.o` 的话** ，只需要加上参数 `-c` 即可：
```bash
g++ header.cpp main.cpp 
```
会自动命名生成 `header.o` 和 `main.o`，而不直接生成 `main.exe`。

而后，执行如下命令，即可把`header.o` 和 `main.o` 链接为 `main.exe`
```bash
g++ header.o main.o -o main
```






# 编译器命令行选项

| 类别         | 选项示例                                                       | 说明                  |
| ---------- | ---------------------------------------------------------- | ------------------- |
| **语言标准  ** | `-std=c++11` / `-std=c++17` / `-std=c++20`                 | 指定使用的C++标准版本。       |
| **优化级别**   | `-O0`（无优化）  <br>`-O2`（常用优化）  <br>`-Os`（优化体积）               | 控制编译器优化程度。          |
| **调试信息**   | `-g`                                                       | 生成调试信息，便于使用gdb等调试器。 |
| **警告控制**   | `-Wall`（常用警告）  <br>`-Wextra`（更多警告）  <br>`-Werror`（将警告视为错误） | 帮助发现代码中潜在的问题。       |
| **宏定义**    | `-DDEBUG`  <br>`-DNDEBUG`                                  | 在命令行定义宏，可用于条件编译。    |
| **链接库**    | `-lm`（链接数学库）  <br>`-lpthread`                              | 链接所需的第三方库或系统库。      |
| **输出文件**   | `-o output_name`                                           | 指定生成的可执行文件或库的名称。    |
| **包含路径**   | `-I/path/to/include`                                       | 添加头文件搜索路径。          |
| **库路径**    | `-L/path/to/lib`                                           | 添加库文件搜索路径。          |
